# Implementation Plan: File Original TC for Override-Aware Relink & Decode

**Branch**: `009-drp-importer-must` | **Date**: 2026-04-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/009-drp-importer-must/spec.md`

## Summary

Add `file_original_timecode` to media rows for master clips with a DaVinci Resolve "Set Timecode" override. The relinker matches candidates against both the displayed TC (`start_tc_value`) and the file's original container TC (`file_original_timecode`). At playback time, EMP's new `set_tc_origin_override` setter replaces the probed container TC with the media row's `start_tc_value`, so decode arithmetic lands on the correct file frame. Closes the last relink gap for ~12 VFX clips in the user's gold master.

## Technical Context

**Language/Version**: Lua (LuaJIT) + C++ (Qt6)
**Primary Dependencies**: EMP (editor_media_platform), SQLite, ffprobe (TC probing), dkjson (JSON)
**Storage**: SQLite `.jvp` project files — new field in existing `metadata` JSON blob (no schema change)
**Testing**: LuaJIT tests (`tests/test_*.lua`), integration tests via `--test` mode, `make -j4`
**Target Platform**: macOS (Darwin)
**Project Type**: Single (desktop NLE)
**Performance Goals**: No hot-path regression — setter is one-time per MediaFile open
**Constraints**: Fail-fast asserts, no fallbacks, MVC, TDD mandatory
**Scale/Scope**: ~12 VFX clips affected in production project; camera footage behavior unchanged
